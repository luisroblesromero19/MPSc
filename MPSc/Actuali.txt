Quiero refactorizar mi proyecto MPSc (Modelo Predictivo de Scouting) para comenzar la transición desde una aplicación de cálculo puntual hacia una plataforma de scouting con almacenamiento histórico en MySQL.


CONTEXTO ACTUAL
---------------
El proyecto actualmente:
- Está escrito en Python
- Tiene arquitectura modular
- Tiene modelos Jugador, Club y Resultado
- Calcula scores de compatibilidad entre jugador y club
- Usa una UI web y/o formularios para ingresar datos manualmente
- Actualmente NO usa persistencia histórica
- Los datos viven temporalmente durante la ejecución


OBJETIVO DE ESTA PRIMERA FASE
-----------------------------
Implementar persistencia MySQL para jugadores y temporadas, comenzando por un CRUD completo de jugadores.


IMPORTANTE:
NO modificar todavía la lógica matemática del sistema.
NO eliminar funcionalidades existentes.
La prioridad es integrar persistencia de datos manteniendo compatibilidad con el sistema actual.
